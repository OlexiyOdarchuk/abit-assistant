## `pkg/parser/source.go`

```go
type Source interface {
    Parse(ctx context.Context, programURL string) (*abit.Program, error)

```

Інтерфейс для джерел даних про конкурсну програму. Концепція: osvita зараз єдиний, edbo чекає 2026, тести можуть мокати.

## `pkg/parser/osvita/`

Парсер vstup.osvita.ua. Найбільш стабільне джерело.

### Як працює

1. **Static page**: `GET /y<рік>/r<регіон>/<uid>/<sid>/` → HTML 60-70 KB
   - Назва ВНЗ, програма, spec_code → з `.page-vnz-detail-title h{1,2}`
   - Volume (ліцензований обсяг, квоти, статистика) → з **`<table><tr><td>k</td><td>v</td></tr>`** (це знайшли в коммі `304ecd3` — раніше шукали `<b>` тегі, не спрацьовувало)
   - JS-vars (`statuses`, `rec_types`, `subjects`, `nmts`, `sub4ar`, `k4max`, `eb`, `okr`, `rk`) — з inline `<script>` через `extractJSConfig` (word-boundary regex + balanced-bracket extractor)

2. **API requests**: `POST https://vstup.osvita.ua/api/` з payload `{action: "requests", y, sid, uid, last}` → JSON `{url: "<signed_json_url>"}`. Потім `GET <signed_json_url>` → `{requests: [...], requests_subjects: {...}}`. Fan-out на 8 workers, stride = 500*8.

### Особливості

- **Warm-up**: перший POST часто повертає порожньо (cookie ще не сетнули). Тому `fetchJSONURL` з last=0 викликається до fan-out, just to seed jar.
- **Retry on 429/5xx** з expo backoff (200ms → 400ms → 800ms), max 3 спроби.
- **Cancel-on-error**: коли один worker впав з non-retriable error, `cancel()` каскадує на всіх — щоб не довбати osvita 60 секунд від решти.
- **Незалежні lane-и** (коміт `acc15e1`): кожен worker зупиняється на ВЛАСНІЙ першій порожній сторінці й НЕ обрізає інших. Раніше був cross-lane ratchet (`stopAt = min(...)`), але transient-порожня відповідь в одному lane могла тихо втратити дані іншого. Ціна — до `workers-1` зайвих порожніх fetch на справжньому хвості.

### URL-структура (deep research 2026-06-13)

`https://vstup.osvita.ua/y<рік>/r<регіон>/<uid>/<sid>/`

- `y2025` — рік кампанії; `<sid>` (остання цифра, напр. 1471029) = **ID
  програми/конкурсної пропозиції**; `<uid>` (напр. 282) = **код закладу**
  (НЕ спеціальність). Назви `sid`/`uid` у коді історичні — regex тягне дві
  цифри коректно, значення не хардкодяться.
- ⚠️ **`r<N>` — код ОБЛАСТІ. r14 = Львівська (НЕ Харків!), r21 = Харківська,
  r10 = Івано-Франківська.** Сегмент регіону парсер ігнорує (`[^/]+`), тож
  на роботу не впливає — але приклад у промптах історично «r14» = Львів.

### Browse-endpoint `/spec/` — розблоковує реверс-пошук (фіча №2)

osvita МАЄ сторінку переліку програм за фільтрами:
`https://vstup.osvita.ua/spec/<A-B-C>/<6-слотів>/`

- `/spec/0-0-0/` — база (рівень-регіон-галузь-спеціальність-форма).
- `/spec/1-40-1/0-0-2548-0-0-0/` — Комп'ютерні науки, ~370 програм.
- `/spec/1-40-1/21-0-0-0-0-0/` — фільтр по регіону 21 (Харків).
- `/spec/1-520-1/0-164-2532-0-0-0/` — інша спец.
- Спостереження: у 2-й групі слот 1 = регіон (21=Харків), слот 3 =
  код спеціальності (2548=КН). Точне відображення всіх слотів — **open
  question**, треба зняти з живого `/spec/0-0-0/` на старті кампанії.
- Для фічі №2: ітерувати сторінки `/spec/` за (регіон, спеціальність) →
  зібрати program URL → прогнати кожен через наявний Analyze.
- ⚠️ Перший рефутнутий варіант: NIE `/spec/N-M-K/` = регіон-спец-рівень
  (0-3 голоси проти). Не вгадувати слоти — мапити з живих даних.

## `pkg/parser/abitpoisk/`

Пошук конкретного абітурієнта на abit-poisk.org.ua.

### Особливості

- `POST /api/statements` з `search=<name>` → JSON `{html: "<table>...</table>"}`
- **Broken cert chain**: abit-poisk віддає incomplete TLS chain. `WithInsecureTLS()` як opt-in option. У CLI ввімкнено за замовчуванням з warning.
- `WithInsecureTLS()` тепер мутує існуючий transport, не replace client (коміт `f6192b7`) — щоб не губити CookieJar з попередніх опцій.

## `internal/service/`

Application layer. Компонує парсери + storage у use cases.

### `ProgramService`

```go
Fetch(ctx, url)   *Program       // cache-aware
Refresh(ctx, url) *Program       // bypass cache
FetchDecoded(ctx, url) []Abiturient  // Fetch + abit.Decode
```

**singleflight**: концурентні `Fetch(url)` для тієї самої URL → один реальний запит до osvita. Інші чекають його результату. Інакше viral-share = 50 паралельних парсингів. Коміт `3363e59`.

### `ApplicantService`

```go
Search(ctx, name) []ApplicantEntry  // cache-aware
Refresh(ctx, name) []ApplicantEntry // bypass cache
```

**ErrNoData НЕ кешується**. Якщо людина щойно з'явилась на abit-poisk, не хочемо 24 години показувати "не знайдено".

### `EnrichService` (поки не використовується)

```go
Enrich(ctx, abits) []EnrichedAbiturient
```

Паралельний lookup кожного абіт через ApplicantService. Worker pool capped (default 4). Призначений для майбутнього "show me the 50 most-likely competitors with their other applications".

Готовий, але не wired у handler — Telegram'у важко рендерити 50 рядків з historу одночасно. Залишимо для аналітичного режиму у server-frontend (якщо колись).
