# 02 — Доменна модель

Все в `pkg/abit/`. Pure Go, без I/O. Цей пакет тестуємо найретельніше — це серце.

## Основні типи

### `Abiturient` — один абітурієнт після decode

```go
type Abiturient struct {
    ID             int
    Num            int             // позиція у списку osvita
    Priority       int             // 1..N, де 1 = ця спеціальність — основна
    OtherReq       int             // інший пріоритет, якщо РЕКОМЕНДУЮТЬ туди
    Name           string          // "Іваненко І.О." або "Іва###" якщо приховано
    Status         string          // "До наказу (бюджет)", "Деактивовано", ...
    Quotas         []string        // QuotaKV1, QuotaKV2, QuotaKV3, QuotaSB
    Coefficients   []string        // CoefGK, CoefSK, CoefPCHK, CoefOL, CoefKR, CoefRK
    RecType        string          // "За квотою 1" і т. п.
    Score          float64         // конкурсний бал з osvita
    Documents      bool            // оригінали подані
    StateEducation bool            // подавався на бюджет (true) чи контракт (false)
    CalcLink       string          // згенероване посилання на калькулятор osvita
    AbitLink       string          // посилання на abit-poisk пошук цієї людини
    DetailScores   map[string]float64  // {"Українська мова": 175, ...}
}
```

### `Program` — конкурсна пропозиція з osvita

```go
type Program struct {
    UniversityName, ProgramName, SpecCode string

    // Конфіг з JS-vars на сторінці osvita:
    EB     int                // 40 = на основі ПЗСО (повна загальна сер.)
    OKR    int                // 1 = бакалавр
    K4Max  float64            // макс. коеф 4-го предмета (≤ 0.35 типово)
    RK     float64            // регіональний (default 1.0 якщо нема)
    NMTs   []int              // program-local subject IDs що рахуються як НМТ
    Sub4ar []int              // program-local subject IDs дозволених як 4-й
    Subjects []SubjectMeta    // повний rubric

    // Зскрейпене з таблиці (один <tr><td>k</td><td>v</td></tr>):
    Volume map[string]string  // "Максимальний обсяг державного замовлення": "28", ...

    // Метадані для decoder:
    Statuses, RecTypes map[string]string

    // Сирий список + per-applicant subject scores:
    Requests        []RawRequest        // []any (індекси у decoder.go)
    RequestSubjects map[string]ApplicantSubjects  // applicant_id → subject_id → [score, bonus, intl]
}
```

### `Analysis` — результат прогону через heuristic

```go
type Analysis struct {
    UserScore        float64
    BudgetTotal      int          // ліцензований обсяг (з Volume)
    Quota1Total      int
    Quota2Total      int
    RemainingSpots   int          // вільних на загальному
    AlreadyEnrolled  int          // на наказі / рекомендовано
    CompetitorsTotal int          // ті що пройшли IsCompetitor
    MyRealRank       int          // 1-based проти справжніх
    Chance           ChanceLevel  // High/Medium/Low/Zero/HighQuota1/2/Unknown
    Advice           string       // "Ти 12-й з 18 місць. Шанси чудові!"
    Warnings         []string     // "license-volume-missing", "region-coef-unavailable"
}
```

## Ключові функції

| Файл | Що робить |
|---|---|
| `decoder.go` | `Decode(*Program) []Abiturient` — port `decoder.py`. Розбирає `RawRequest` за фіксованими індексами (rawIdx*). |
| `filter.go` | `IsCompetitor(ab, score)` — `>=` (tie = competitor, як в Python). `IsCompetitorWith` — з manual override map. |
| `analyze.go` | `Analyze(prog, abits, in) Analysis` — port `recalculate_analysis`. Розкладає по квотах, рахує rank+chance. |
| `calculator.go` | `ComputeRating(prog, in) float64` — port `calculate_rating_score`. Weighted average (як Python), не raw sum. |
| `stats.go` | `Summarize`, `Distribution(bucketSize)`, `RankByScore`, `Percentile`, `RealCompetitors` |
| `links.go` | `GenerateAbitPoiskLink(name)`, `GenerateCalcLink(subj, score, eb, okr)` |
| `program.go` | `Program.BudgetVolume() / Quota1Volume() / Quota2Volume()` — substring match у Volume |
| `codes.go` | Constants: QuotaKV1..QuotaSB, CoefGK..CoefSB, RequiredSubjects, MaxProfileSubjects, CreativeContest |
| `types.go` | Самі struct + RawRequest/SubjectScore type aliases |

## Найважливіше для майбутнього себе

1. **`IsCompetitor` — heuristic-based** (контракт-only / "деактивовано" / "до наказу" / score). Не магія. Python робить додатковий abit-poisk-lookup для priority>1 — у нас це винесено в `EnrichService` (поки не використовується в боті).

2. **`ComputeRating` — weighted average**, не raw sum. Бо Python робить `sum_scores / sum_coeffs`. Якщо коефіцієнти нормовані (sum=1) — обидва формули збігаються; інакше weighted нормалізує до 100..200 шкали.

3. **`Analyze` — нечесно з невідомим volume** не може. Якщо `BudgetVolume() == 0` — повертає `ChanceUnknown` + `"license-volume-missing"` у Warnings. Раніше fabricated `budget = CompetitorsTotal + 1` що давало майже всім High → виправлено.

4. **`Volume` парситься з `<table><tr><td>k</td><td>v</td></tr>`** — не з `.block-pro-vnz <b>` як думали спочатку. Перевірити це для нової кампанії 2026 — структура HTML може змінитись.

5. **`Quotas` / `Coefficients` — `[]string` з constants з codes.go**. Не голий string. `slices.Contains(ab.Quotas, abit.QuotaKV1)` замість regex.

## Чого тут НЕ повинно бути

- Жодних типів про бот, Telegram, FSM, БД — це presentation/persistence
- Жодних мережевих викликів — це у `pkg/parser/*`
- Жодного `log.Info(...)` — pure-функції не логуються (caller вирішує)
