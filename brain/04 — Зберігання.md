# 04 — Зберігання

## Стек

- **SQLite** через `modernc.org/sqlite` (pure-Go, без CGo)
- **sqlc** для генерації type-safe запитів з SQL
- **Embed migrations** через `//go:embed migrations/*.sql`
- **JSON-helpers** поверх sqlc для blob-полів (NMT scores, settings, Program snapshots)

## Чому без CGo

`mattn/go-sqlite3` швидше, але потребує CGo → `scratch` Docker не виходить, треба `alpine` базу. `modernc.org/sqlite` — pure-Go, тому фінальний образ ~13 МБ замість ~50 МБ. Для бота, який не робить мільйонів запитів — різниця у швидкості несуттєва.

## Шлях до БД

`internal/config/config.go::defaultDatabasePath()` резолвить так:

1. **Legacy**: якщо `./data/abit.db` існує в cwd — береться (для існуючих installs)
2. **XDG**: `$XDG_DATA_HOME/abit-assistant/abit.db`
3. **Home**: `$HOME/.local/share/abit-assistant/abit.db`
4. **Last resort**: `./data/abit.db`

`DATABASE_PATH=...` env override все. Docker compose явно задає `/data/abit.db` + bind-mount `./data:/data`.

**Чому це важливо** (коміт `f8eb3f2`): раніше default був `./data/abit.db`. Під час smoke-тестів `rm -rf data/` зносив юзерські дані. XDG default + startup-log з absolute path + user count → видно одразу, що відкрили.

## Схеми (migrations)

Усі в `internal/storage/migrations/`:

### `0001_init.sql`

```sql
users
├── tg_id              INTEGER PK (natural, не surrogate `id` як в Python)
├── nmt_scores         TEXT NOT NULL DEFAULT '{}'    -- JSON
├── settings           TEXT NOT NULL DEFAULT '{}'    -- JSON (UserSettings)
├── activates          INTEGER NOT NULL DEFAULT 0
├── right_activates    INTEGER NOT NULL DEFAULT 0
├── created_at         INTEGER NOT NULL DEFAULT unixepoch()
└── updated_at         INTEGER NOT NULL DEFAULT unixepoch()  -- Python не мав

saved_lists
├── id                 INTEGER PK
├── user_tg_id         INTEGER FK users(tg_id) ON DELETE CASCADE
├── name               TEXT
├── url                TEXT
├── data               TEXT NOT NULL           -- JSON Program
├── created_at         INTEGER NOT NULL DEFAULT unixepoch()
└── (share_token додано в 0003)

program_cache
├── url                TEXT PK
├── data               TEXT NOT NULL           -- JSON Program
└── updated_at         INTEGER NOT NULL DEFAULT unixepoch()

applicant_cache
├── name               TEXT PK
├── data               TEXT NOT NULL           -- JSON []ApplicantEntry
└── updated_at         INTEGER NOT NULL DEFAULT unixepoch()
```

Усі STRICT (SQLite ≥ 3.37). INTEGER unix epoch для часу — порівнюємо проти `time.Now().Unix()`.

### `0002_bot_fsm.sql`

```sql
bot_fsm
├── tg_id      INTEGER PK FK users(tg_id) ON DELETE CASCADE
├── state      TEXT NOT NULL DEFAULT ''
├── data       TEXT NOT NULL DEFAULT '{}'  -- JSON {key: value}
└── updated_at INTEGER NOT NULL DEFAULT unixepoch()
```

### `0003_share_tokens.sql`

```sql
ALTER TABLE saved_lists ADD COLUMN share_token TEXT NOT NULL DEFAULT '';
UPDATE saved_lists SET share_token = lower(hex(randomblob(16))) WHERE share_token = '';
CREATE UNIQUE INDEX idx_saved_lists_share_token ON saved_lists(share_token) WHERE share_token <> '';
```

Це security fix: раніше `/start list_<id>` дозволяв енумерувати чужі списки. Тепер `/start share_<token>`, де token — 128 біт rand.

## sqlc

Конфіг у `sqlc.yaml`:
- schema: `internal/storage/migrations/`
- queries: `internal/storage/queries/`
- output: `internal/storage/db/` (НЕ редагувати руками)
- rename: `url` → `URL`, `tg_id` → `TgID` (Go-конвенція для акронімів)

Команда: `~/go/bin/sqlc generate`.

## Race-safe counters

`IncrementActivates` робить `INSERT ... ON CONFLICT(tg_id) DO UPDATE SET activates = activates + 1` — атомарно. Python мав read-modify-write race на лічильниках.

## JSON-helpers

sqlc повертає `string` для TEXT-полів. `store.go` додає типобезпечну обгортку:

- `UserSettings` struct з полями (Quotas, RegionCoef, CreativeScorePrediction)
- `UserNMT = map[string]float64`
- `SavedList` з `*abit.Program` (не raw JSON)
- `ErrCacheMiss` / `ErrCacheStale` / `ErrNotFound` sentinel errors

`ErrNotFound` — для saved_lists (це не cache); `ErrCacheMiss/Stale` — для program/applicant cache. Розділено в коммі `66b2a08`.

## Concurrency

- `SetMaxOpenConns(1)` — SQLite single-writer, не сенс множинні з'єднання
- `busy_timeout=5000` — модерн-driver сам ретраїть SQLITE_BUSY 5 секунд

## Cache TTLs

| Cache | Default TTL | Чому |
|---|---|---|
| program_cache | 10 хв | osvita.ua оновлюється раз у 1-2 год, але юзер може багато разів запитати ту саму програму |
| applicant_cache | 24 год | Якщо абітурієнт є на abit-poisk, його заяви рідко змінюються щоденно |

Адмін може зробити `🧹 Очистити кеш` у `/admin` — `VacuumCaches(0, 0)` зносить усе.
