# 08 — Roadmap

## ✅ Зроблено (травень 2026)

- Парсер `vstup.osvita.ua` з fan-out + retry/backoff + warm-up cookies
- Парсер `abit-poisk.org.ua` з broken-cert opt-in
- Декодер усіх форм статусів і квот (port `decoder.py`)
- Profile flow (НМТ + квоти + РК + творчий бал) з валідацією 3+1
- Реальний рейтинговий бал (weighted average + RegionCoef)
- Marker 🔴/🟢 у списку + toggle "Тільки конкуренти"
- Деталі абітурієнта + abit-poisk історія
- Persistent FSM у SQLite
- Saved lists з refresh+diff, share via deep-link token, JSON export
- Summary-екран (місце, шанс, вердикт)
- Manual toggle «це не конкурент» з sticky overrides
- Гістограма балів картинкою (PNG через go-chart/v2)
- `/admin` — статистика, broadcast (rate-limited), vacuum cache
- Docker scratch image + Compose (non-root, read-only rootfs)
- AES-CBC формула EDBO підтверджена з production-коду
- Security + correctness аудит (40+ findings, основні закриті)

## ⏸ Чекає 2026 кампанію

### EDBO драйвер (high priority on launch)

→ [[06 — EDBO Research]]

Потребує живого offer URL з даними. Розрахунковий effort: 1-2 дні роботи після того, як перший XHR captured. Реалізація як `internal/parser/edbo/`, плагається у ProgramService як другий Source.

Кроки:
1. Capture живого offer → знайти POST endpoint списку
2. Знайти точну формулу salt (зараз знаємо що `"v" + str(a*b)`, треба знайти конкретні a, b)
3. Знайти декілька URL варіантів (2026 vs архів) — щоб бот фоллбекав між осіта та едбо
4. Тести: real-blob-decode case у `internal/edbo/crypto_test.go`

## ⏸ Чекає реальних користувачів

### `/about` показує VERSION

Зараз `cmd/bot/main.go` має `VERSION` build-arg для Docker, але ніде не показано. Колись додати в about-screen.

### Migration tool

Якщо треба rollback міграцій. Зараз `runMigrations` тільки forward. Поки що нічого не ламали — не критично.

### Tests на bot handlers

Поточні bot-тести покривають тільки рідкі речі (NMT subject grid duplicates fix). Більшість handlers зломається не від rev-engineering edge case, а від API-changes у telebot/v3 — а ці changes ловить `go test` через build error.

Якщо колись Bot CI: моки `tele.Context`, перевіряти `c.Send` / `c.Edit` calls.

## 🎯 Великі фічі — «максимально спростити життя вступнику»

Три ідеї, що рухають продукт від «показати список» до «сказати, що робити».
Перша й третя не потребують EDBO — будуються на наявних osvita + abit-poisk.

### 1. Симуляція пріоритетів («зняти з конкуренції тих, хто піде деінде»)

**Чому головне:** це РІВНО та діра точності, що зафіксована в
[[10 — Правила вступу і формули#Що евристика бота НЕ моделює (відомі межі точності)|10 — межі точності, п.1]].
Адресне розміщення рекомендує кожного на РІВНО одну бюджетну програму —
його найвищий пріоритет, де він проходить. Тож конкурент, що стоїть вище
за мене, але ця програма в нього пріоритет 2/3, і він проходить на свій
пріоритет 1 — фактично НЕ займає тут місце. Знявши таких, отримуємо
реальний прохідний.

**Інфра вже є:** `EnrichService` (fan-out abit-poisk, 4–8 воркерів, кеш 24h)
написаний і не використовується — це його робота.

**Механіка (евристика, не повний глобальний solve):**
1. Беремо конкурентів ВИЩЕ юзера з пріоритетом ≥ 2 на цій програмі.
2. abit-poisk по імені → інші заяви + їх пріоритети + **статуси**.
3. **Дешевий сигнал:** якщо abit-poisk уже показує «Рекомендовано / До
   наказу» на програмі з вищим пріоритетом — конкурент знятий, без
   повторного аналізу. Дорогий шлях (рекурсивно аналізувати ту програму)
   потрібен рідше.
4. Перерахувати rank без знятих → чесніший шанс + лейбл «−N піде деінде».

**Граблі:** масковані імена (`Іва###`) не пробити; колізії однофамільців;
rate-limit abit-poisk (тому scope = лише конкуренти вище юзера, не весь
список); off-season abit-poisk порожній — реальні дані з липня 2026.

**Effort:** 2–3 дні. Найбільший приріст точності з усього беклогу.

### 2. Режим «куди я поступлю» (реверс-пошук за спеціальністю + областю)

Юзер задає бажану спеціальність + ±області → бот знаходить усі програми
галузі в тих областях і рахує шанси на кожній, сортує reach/match/safety.

**Endpoint ЗНАЙДЕНО + enumeration parser ГОТОВИЙ** (2026-06-13). Слоти
`/spec/` розшифровано на живих даних, парсер `SpecFilter` +
`Parser.BrowsePrograms` написаний і протестований на фікстурі (коміт
`2a9b6dc`). Деталі → [[05 — Парсери і сервіси#Browse-endpoint `/spec/` — розблоковує реверс-пошук (фіча №2)|05 — browse-endpoint]].

**Готово:** `Parser.BrowsePrograms` + `FetchUniversities`/`MatchUniversity`
(коміти `f4b9c79`, `589d6af`); `service.DiscoverService.WhereCanIGetIn` —
browse → Fetch → ComputeRating → Analyze → ранжування за шансом, cap +
Found, drop-failures (коміт `74c6016`, з тестами).
**Лишилось для №2:** (1) bot UX — вибір галузі (11, статично зі `<select>`)
+ регіонів + бюджет-тоглу, рендер ранжованого списку з пагінацією;
(2) опційно: каскадний AJAX спеціальностей для дрібнішого фільтра.
**Effort:** ~1 день (переважно bot UX).

### 3. Оптимізатор списку пріоритетів («як розставити мої 5 заяв»)

Природне продовження №1. Дано 5 бюджетних програм + бал юзера → симуляція
адресного розміщення підказує оптимальний порядок пріоритетів, щоб
максимізувати шанс потрапити на найбажанішу. Класифікація reach / match /
safety, щоб не злити всі пріоритети на reach. Будується на №1 + Saved lists
(інфра порівняння вже є, [[#Cross-program compare]]).

**Effort:** після №1 — ще 2 дні.

## 💭 Можливо колись

### Server-frontend

У `AbitAssistant-3.0.md` була ідея Wails-десктоп + сайт-frontend. Це окремий проєкт, не пріоритет. `internal/abit` уже як бібліотека — будь-хто може взяти.

### CLI-tool публічний

`cmd/cli` зараз dev-тільки. Якщо колись юзер хоче скриптити аналіз свого бал — упакувати як Homebrew/AUR пакет.

### Multi-source enrichment

`EnrichService` готовий, але не використовується. Якщо колись зробити аналітичний режим "покажи мені всю історію 50 топ-абіт" — там і ввімкнути.

### Чарти крім гістограми

Зараз тільки розподіл балів. Можна додати:
- Динаміка кількості заяв у часі (потребує snapshots)
- Heat map спеціальностей по регіонах (агрегувати кілька programs)

### Cross-program compare

«Зберегти 5 програм, порівняти». Saved lists інфраструктура є — додати порівняльну в'ю.

## ❌ НЕ будемо робити

- **Не парсимо vstup.edbo.gov.ua headless-браузером** — це повільно і легко тригериться anti-bot. Тільки прямі XHR через `net/http`.
- **Не зберігаємо PII довше ніж треба** — кеші TTL, saved-lists по share-token (opaque).
- **Не push-сповіщення про зміну балу** — це постійний polling osvita, etyka сумнівна, навантаження зайве.
- **Не платні features** — GPLv3, проект open source.
