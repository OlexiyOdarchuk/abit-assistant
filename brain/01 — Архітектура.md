# 01 — Архітектура

## Standard Go Layout

```
abit-assistant/
├── cmd/                ← точки входу, лише wiring
│   ├── bot/main.go        Telegram-бот
│   └── cli/main.go        dev CLI (aa osvita | abitpoisk | decode | edbo)
│
├── pkg/                ← публічне ядро, `go get`-абельне
│   ├── abit/              домен: types, decoder, filter, stats, calculator, links, analyze
│   └── parser/            інтерфейс Source + реалізації
│       ├── osvita/           vstup.osvita.ua (JSON API + JS-vars зі сторінки)
│       └── abitpoisk/        abit-poisk.org.ua (broken-cert opt-in)
│
├── internal/           ← приватна реалізація
│   ├── bot/               Telegram handlers, FSM, callback codec, middleware
│   │   ├── callback/        typed args для callback_data
│   │   └── fsm/             persistent FSM поверх SQLite
│   ├── service/           use cases: ProgramService, ApplicantService, EnrichService
│   ├── storage/           SQLite + sqlc + embed migrations + JSON helpers
│   │   ├── migrations/      SQL schema
│   │   ├── queries/         SQL queries (sqlc input)
│   │   └── db/              sqlc output (НЕ редагувати руками)
│   ├── config/            env loader + .env
│   ├── visualizer/        go-chart/v2 PNG гістограма
│   └── edbo/              AES-CBC дешифровка ПІБ (для майбутнього edbo-драйвера)
│
├── tools/edbo-reverse/    Playwright скрипти реверсу EDBO (gitignored)
├── brain/                 ← ти тут
├── migrations/            (none — все в internal/storage/migrations/)
└── README.md, .env.example, Dockerfile, docker-compose.yml, sqlc.yaml
```

## Залежності між шарами

Правило: **внутрішнє не залежить від зовнішнього**.

```
cmd/  ─────► internal/{bot,service}
                │
                ▼
              internal/{storage, config, visualizer, edbo}
                │
                ▼
              pkg/{abit, parser/*}        ← pure domain + adapters
                │
                ▼
              stdlib + third-party (telebot, sqlc-generated, modernc-sqlite,
                                    goquery, go-chart)
```

- `pkg/abit/` — pure, no I/O. Можна імпортувати з будь-якого проєкту.
- `pkg/parser/*/` — мережа, але приймає `context.Context`, безпечно до cancel.
- `internal/service/` — компонує parser + storage у use cases. Тільки звідси хендлери бота тягнуть дані.
- `internal/bot/` — лише Telegram presentation, не лізе в parser напряму.

Це Hexagonal / Clean Architecture lite: `pkg/abit` = domain core, `pkg/parser` = ports+adapters, `internal/service` = application, `internal/bot|cli` = presentation.

## Чому Standard Go Layout

Це не моя вигадка — це домовленість community ([`golang-standards/project-layout`](https://github.com/golang-standards/project-layout)). Дає:

- **`internal/`** Go-compiler сам блокує імпорти ззовні модуля → захищає приватну реалізацію
- **`pkg/`** — конвенціональна назва для публічних бібліотек → користувач знає що `go get .../pkg/abit` — це експоновано
- **`cmd/<name>/main.go`** — стандарт для бінарників. Кожен у своєму каталозі, можна build окремо

## Точки входу

### `cmd/bot/main.go`

Збирає граф залежностей (config → storage → parsers → services → bot.Deps) і блокує на `b.Run(ctx)`. На SIGTERM ctx скасовується → `tg.Stop()` → defer закриває store.

### `cmd/cli/main.go`

Dev playground. Корисні команди:
- `aa osvita <url>` — парс + JSON у stdout
- `aa abitpoisk "<name>"` — пошук абітурієнта
- `aa decode` — читає Program з stdin, декодує
- `aa edbo decrypt <b64> <salt> <year>` — низькорівневий тест AES

## Ключові архітектурні рішення

| Рішення | Чому |
|---|---|
| **Go 1.26+** | telebot/v3, generics, slog, slices.Contains, range over int — все нативно |
| **modernc.org/sqlite** | Pure-Go, без CGo → scratch Docker. Pure-Go повільніше за mattn/go-sqlite3, але це не вузьке місце |
| **sqlc** | Type-safe SQL → Go. Кожен запит — це функція, що приймає типізовані параметри. Жодних `*sql.Row.Scan(&v1, &v2)` руками |
| **telebot v3** | Чистий API, middleware chain, inline buttons як `tele.Btn{Unique}` — type-safe dispatch |
| **FSM у SQLite** | In-memory FSM ламається на рестарті. Юзер посеред введення НМТ — перезапуск — продовжує з місця |
| **All-inline UX** | Reply-keyboard mix із inline як у Python = плутанина. Все через inline + `/commands` |
| **singleflight** | Дедуплікація конкурентних `Fetch(url)` — інакше N юзерів = N парсингів osvita |
| **GPLv3** | Як у Python-оригіналі. Open source, навколо нього спільнота |

## Чому не Hexagonal у повному сенсі

Robitи interfaces+adapters для кожного шару — overengineering для бота. `pkg/parser/source.go` має `Source` interface (так osvita/edbo/тести взаємозамінні), але `internal/storage` — це конкретна структура `*storage.Store`, не abstraction. Якщо колись треба буде Postgres — refactor буде, але передчасно це не робиться.
